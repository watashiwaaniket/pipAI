# Contributing to PIP-AI 📟

First off, thank you for considering contributing to PIP-AI! It's people like you that make PIP-AI a better tool for the offline AI community.

## 📜 Code of Conduct

By participating in this project, you are expected to uphold our [Code of Conduct](./CODE_OF_CONDUCT.md).

## 🚀 How Can I Contribute?

### Reporting Bugs
- Use the GitHub Issue Tracker.
- Describe the steps to reproduce the issue.
- Mention your device model and Android/iOS version.

### Suggesting Enhancements
- Open an issue with the tag `enhancement`.
- Describe the feature and why it would be useful for an offline terminal experience.

### Pull Requests
1. Fork the repo and create your branch from `main`.
2. Ensure your code follows the existing terminal aesthetic.
3. If you changed UI, include screenshots in your PR.
4. Make sure your code passes TypeScript checks: `npx tsc --noEmit`.

## 🎨 Design Guidelines
- **Keep it Retro**: Use `#00FF41` (Matrix Green) or Amber for primary text.
- **Mono Only**: All UI text must use `Share Tech Mono`.
- **ASCII Borders**: Use double-line or block-style ASCII for containers.

## 🛠️ Development Environment
Follow the setup instructions in the [README.md](./README.md#developer-setup).
